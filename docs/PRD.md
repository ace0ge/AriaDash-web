# AriaDash — 产品需求文档

## 1. 产品概述

AriaDash 是一款面向 iPhone 的 Progressive Web App (PWA)，用于远程控制 aria2 下载服务。用户通过浏览器即可管理下载任务，无需安装原生应用。

## 2. 目标用户

- 在服务器/ NAS / 本地运行 aria2 的用户
- 希望通过手机远程管理下载任务的用户
- 追求轻量、无后端、隐私友好的用户

## 3. 用户故事

### 3.1 首次使用

> 作为一个新用户，我希望能配置我的 aria2 服务器连接，以便开始管理下载。

- 输入 RPC 地址、端口、Secret Token
- 点击"测试连接"验证配置是否生效
- 配置自动保存到 localStorage，下次打开无需重复输入

### 3.2 查看下载列表

> 作为一个用户，我希望查看所有下载任务的状态，以便了解下载进度。

- 显示所有任务列表（进度条、速度、ETA、文件大小）
- 按状态筛选：活跃 / 等待 / 已完成 / 已停止 / 全部
- 实时更新进度（WebSocket 推送或轮询）

### 3.3 快捷操作（手势）

> 作为一个用户，我希望通过滑动手势快速操作单个任务，无需进入详情页。

- **左滑卡片** → 显示"暂停/恢复"操作面板，松手即执行
- **右滑卡片** → 显示"删除"操作面板，松手即执行
- 卡片滑动时视觉跟随手指位移，有回弹动画

### 3.4 批量管理下载任务

> 作为一个用户，我希望同时选中多个任务进行批量操作，以提高效率。

- 点击 Header 右上角 ☑ 按钮进入"多选模式"
- 左侧出现复选框，点击选中/取消选中单个任务
- 底部弹出批量操作栏：暂停选中 / 恢复选中 / 删除选中
- 点击"取消"退出多选模式
- 多选模式下不可滑动操作

### 3.5 任务详情

> 作为一个用户，我希望查看任务的完整信息，并执行详细操作。

- 点击卡片进入详情页
- 显示文件大小、速度、ETA、连接数、协议、错误数等完整信息
- 暂停 / 恢复 / 删除操作按钮在详情页内
- 右滑左侧边缘返回列表（iOS 原生返回手势风格）
- 左上角 ← 返回按钮作为备选

### 3.6 添加新下载

> 作为一个用户，我希望添加新的下载任务，以便下载文件。

- 支持 HTTP / HTTPS URL
- 支持 Magnet 磁力链接
- 支持上传 .torrent 种子文件
- BT 任务可选择下载的文件

### 3.7 仪表盘概览

> 作为一个用户，我希望看到整体下载状况，以便了解系统状态。

- 全局下载速度 / 上传速度
- 活跃任务数 / 等待任务数 / 已完成任务数
- 总下载量 / 总上传量

### 3.6 PWA 体验

> 作为一个 iPhone 用户，我希望将应用添加到主屏幕，以获得接近原生 App 的体验。

- 添加到主屏幕后全屏运行（无 Safari 地址栏）
- 有独立的应用图标和启动画面
- 离线时显示友好提示
- 支持深色模式

## 4. 功能列表

- [x] 服务器配置（地址、端口、Secret 输入 + 连接测试）
- [ ] 下载列表展示（含进度、速度、状态、ETA）
- [ ] 按状态筛选任务
- [ ] 添加下载（URL / Magnet）
- [ ] 暂停 / 恢复 / 删除任务

## 5. 非功能需求

| 需求 | 要求 |
|------|------|
| 首屏加载 | < 3s (3G)，< 1s (WiFi) |
| 离线表现 | 配置页可离线打开，显示友好的离线提示 |
| iOS 兼容 | Safari on iOS 16+ |
| 安全性 | Secret Token 仅存 localStorage，不上传、不暴露 |
| 可部署性 | 纯静态文件，可部署至 GitHub Pages / Vercel / Netlify |
| 体积 | 构建产物 < 500KB (gzip) |

## 6. 约束

- 无后端服务，纯浏览器端
- 必须兼容 aria2 的 CORS/WebSocket 限制（需要服务端配合 `--rpc-allow-origin-all`）
- 不存储用户配置到云端
- 不开屏广告、无追踪代码

## 7. 成功指标

- 用户从打开页面到首次成功连接到 aria2 不超过 30 秒
- 核心操作（暂停/恢复/删除）的完成时间 < 1 秒
- PWA Lighthouse 评分 > 90
